Background

Sensors are used to grab environment input and process them into meaningful output. Barcodes are a common representation of data in the real world, and barcode readers are a very effective and efficient way to transfer physical information into digital data.

There are two types of barcode sensors - pen-type scanners and laser scanners. Both operate in a similar way - they use reflected light values and scan across the barcode at the same speed, to determine the widths of the bars and gaps.

Project Aim

Develop code to implement a simple barcode reader. The code should reliably read simple and relatively large bars in a straight line - and should be able to do so with different barcodes.

Project Specification

1. Robot

  • You must build your own robot. You should aim to finish your robot construction by the end of Week 8. This sample demo bot construction can be used.

  • The robot should move along the barcode and use a light sensor to obtain reflected light levels.

  • The robot has two wheels, left connected to port A and right connected to port B.

  • The robot has two light sensors, left connected to port 1 and right connected to port 2.

2. Barcode

  • Barcodes have three types of codes: a start code, an end code and number codes.

  • The start and end codes are 30mm of black squares.

  • There are three types of number codes, 1, 2 and 3. These are represented by different configurations of 30mm black and white strips.

  • All number codes end with a 10mm white strip.

  • A special number code, 4, is represented by a harder configuration.

3. Output

Your robot should ignore the start code, travel along the barcode recording codes, and stopping after the end code is detected. Barcodes can be of any length.

When you pass each code (group of 4 cells), you must beep or indicate a number has been recorded. You can also indicate by changing the printed number, or printing something temporarily on the screen. You do not need to make sounds or print anything for the start code (unless you wish to).

When you reach the end code, you must stop the robot and optionally print the result on the screen. If you do not print out values onto the screen, the beeps must be different.
