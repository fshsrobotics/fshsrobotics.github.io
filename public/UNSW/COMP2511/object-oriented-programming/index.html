1.2 OOP and ADTs

  • Abstract data type: are like stacks, queues, lists - they store data and provide an interface to store it

  • Object oriented programming: Instead of figuring out at what data to store, figure out what objects make up a system

    • e.g. cars in a simulation, cars with attributes speed and location (corresponding to its state)

    • Methods: functions like moving location

  • Encapsulation: distributing the flow of control to different parts of the system

    • e.g. the simulation cannot see the location of each car, but it can call moveCar

    • In turn, the car can move based on its own location

  • OOP and ADTs are different. In OOP:

    • Processing can be more distributed, but harder to access multiple data items (e.g. collision between two cars is difficult to implement)

    • Allow multiple class implementations (like a graph having both matrix and list), and easier to extend classes

    • Harder to verify correctness

    • Dynamic Binding
Employee e = new Manager("Brad", 100000);
e.toString();

    • Will print out that it is a manager

    • However, Brad is treated just as an Employee (unless overriden)

    • No need to cast - only cast when doing equality and cloning

1.3 Liskov Substitution Principle

  • Liskov Substitution Principle: s is a subclass of t if every method of s can be defined in terms of methods of *t*

    • All methods from a class 'should work' in a subclass

    • Employee's getSalary method should work with Managers

    • Any constraints in Employee (such as 'salary can only go up') should also apply to Manager

  • All squares are rectangles, so it is possible thatSquare extends Rectangle

    • However, you should check that all constraints are respected and consider its operations (methods) as well

    • setWidth(5) in a Square may automatically make it a square and set its height to 5 as well, but consider when this happens:

Rectangle r = new Square(3);
r.setWidth(5); // is the height 3 or 5?
System.out.println(r.getArea()); // is it a Rectangle or a Square?

  • We expect that the area is 5 * 3, and this does not hold for Square

  • The behaviour of a Square is not the same as a Rectangle, so Square should not extend Rectangle

public boolean equals(Object other) {
    if (this == other) return true;
    if (other == null) return false;
    if (getClass() != other.getClass()) return false;
    Employee o = (Employee) other;
    return name.equals(o.name) && salary == o.salary;
}

public boolean equals(Object other) {
    if (!super.equals(other)) return false;
    Manager m = (Manager) other;
    return hireDate.equals(m.hireDate);
}
